# SaaS Setup (College Subscriptions)

This project now supports college-level SaaS subscriptions with plan limits and Razorpay upgrades.

## Plans

- `free`: 100 students, 5 jobs, analytics disabled
- `basic`: 500 students, 20 jobs, analytics enabled
- `pro`: unlimited students/jobs, analytics enabled
- `enterprise`: unlimited students/jobs, analytics enabled

Plans are auto-upserted via `ensureSaasPlans()` when onboarding, billing, or limit checks run.

## Required Env Vars (Razorpay SaaS)

- `RAZORPAY_KEY_ID`
- `RAZORPAY_KEY_SECRET`
- `RAZORPAY_WEBHOOK_SECRET` (shared fallback)
- `RAZORPAY_SAAS_WEBHOOK_SECRET` (recommended for SaaS webhook)
- `RAZORPAY_COLLEGE_BASIC_PLAN_ID`
- `RAZORPAY_COLLEGE_PRO_PLAN_ID`
- `RAZORPAY_COLLEGE_ENTERPRISE_PLAN_ID`

Important: webhook secret must be the secret string generated by Razorpay webhook settings, not your webhook URL.

## Key Routes

- College onboarding (public): `/onboarding/college`
- TPO billing: `/admin/billing` (alias: `/tpo/billing`)
- SaaS create subscription API: `POST /api/saas/razorpay/create-subscription`
- SaaS webhook API: `POST /api/saas/razorpay/webhook`
- Platform admin: `/super-admin`

## Enforcement

- Student creation limit enforced in `actions/register.ts`.
- Job posting limit enforced in `actions/post-job.ts`.
- Analytics access enforced in `app/admin/analytics/page.tsx`.
- College suspend state enforced at login/middleware.
